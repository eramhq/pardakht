# الگوریتم‌ها و منابع

## الگوریتم Luhn

کلاس `CardNumber` از [الگوریتم Luhn](https://en.wikipedia.org/wiki/Luhn_algorithm) (ISO/IEC 7812-1) برای اعتبارسنجی شماره کارت‌های بانکی ایرانی استفاده می‌کند. این همان الگوریتم چک‌سام است که در سراسر جهان برای اعتبارسنجی کارت‌های اعتباری/نقدی استفاده می‌شود.

## اعتبارسنجی IBAN (ISO 13616)

کلاس `Sheba` شبای ایرانی را با استفاده از الگوریتم چک‌سام [ISO 13616](https://en.wikipedia.org/wiki/International_Bank_Account_Number#Validating_the_IBAN) mod-97 اعتبارسنجی می‌کند. پیاده‌سازی اعداد بزرگ را با پردازش رشته عددی در بخش‌ها مدیریت می‌کند تا از سرریز عدد صحیح جلوگیری شود.

## شماره‌های شناسایی بانک (BIN)

نگاشت کارت به بانک از جداول BIN (شماره شناسایی بانک) در دسترس عموم برای بانک‌های ایرانی استفاده می‌کند. ۶ رقم اول شماره کارت بانک صادرکننده را شناسایی می‌کند.

## کدهای بانک شبا

نگاشت شبا به بانک از کد بانک تعبیه شده در موقعیت‌های ۵ تا ۷ شبا (بعد از پیشوند IR و ارقام کنترلی) استفاده می‌کند. این کدها توسط بانک مرکزی ایران (CBI) تخصیص داده شده‌اند.

| سامان (سپ) | SOAP | پرداخت الکترونیک سامان |
| پارسیان (پک) | SOAP | تجارت الکترونیک پارسیان |
| سداد | REST | بانک ملی ایران |
| پاسارگاد | REST | بانک پاسارگاد |
| زرین‌پال | REST | زرین‌پال |
| آیدی‌پی | REST | آیدی‌پی |
| زیبال | REST | زیبال |

| نکست‌پی | REST | نکست‌پی |
| وندار | REST | وندار |
